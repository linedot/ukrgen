# Exploring GEMM kernel performance on the SpacemiT k1 (X60)

The X60 core supports RVV 1.0 with a VLEN=256 bit. On [their website](https://www.spacemit.com/en/spacemit-x60-core/) SpacemiT is sparse with details, the most relevant info is:

- 256 bit vector computing bandwidth (assuming this means 1 FU that can do RVV with VLEN=256bit)
- 8-stage dual-issue in-order
- dual-issue of scalar and vector read/write instructions

with VLEN=256bit and one FU, 8 FP64 FLOP/cycle should be the peak. Let's take a look how many independent instructions it takes to hide the fma latency and reach theoretical peak performance:

```
vsetvli t6, zero, e64, m1, ta, ma
.loop:
vfmacc.vf v0, f0, v30
vfmacc.vf v1, f1, v30
vfmacc.vf v2, f2, v30
vfmacc.vf v3, f3, v30
vfmacc.vf v4, f4, v30
vfmacc.vf v5, f5, v30
add t1, t1, -1
bnez t1, .loop
```
Result:
```
Max. 7.99598 FLOP/cycle
```
It's 6. Alright, this is equivalent to a 1vx6 microkernel, let's start introducing load instructions. For now just loading from the same address to ignore memory architecture/hierarchy:
